相關提問:

  1. 在製作 HW1 之前是否實際練習 Tutorial1 與 Tutorial2?A:是
  2. Webots 是什麼樣的軟體？它主要用途為何？A:用於各種機器人應用，包括輪式機器人、工業手臂、飛行無人機、水下機器人、多足機器人、自動駕駛車輛等。目前在課程中，主要用於機械結構運動模擬。
  3. 在 Tutorial1 中，學習者需要操作的主要機器人模型名稱是什麼？A:Robot
  4. 如何在 Webots 中建立並執行一個新的模擬世界（world）？A:

    建立新專案：在頂部選單中選擇 Wizards（或「嚮導」）→ New Project Directory（「新專案目錄」），依照指示輸入專案名稱與儲存路徑。

    建立新世界：在專案目錄下，自動會建立一個名為 worlds 的資料夾，並生成一個新的 .wbt 檔案，這就是你的模擬世界檔案。

  6. Tutorial1 中，學習者需要修改哪些參數或檔案來控制機器人的行為？A:1.電機時間步長 2.電機目標位置 
  7. 請簡述 Webots 控制器（controller）的功能及其與機器人的關係。A:為robot中的device提供控制程式來控制機構旋轉及作動的控制。
  8. 在 Tutorial2 中，機器人需要完成什麼樣的任務或行為？A:創建一個與環境互動的球。我們將處理與節點相關的幾個概念：它們的含義是什麼，如何創建它們，它們如何必須隸屬等等。此外，我們還將看到如何設置物理效果。
  9. Webots 中的 "Supervisor" 是什麼？它與一般機器人控制器有何不同？A:Supervisor 是一種具備模擬世界全域權限的特殊控制器，與一般機器人控制器相比，它能夠進行更高層次的實驗管理與環境控制，適合用於複雜模擬實驗、自動化測試和數據收集。
  10. 請說明如何在 Webots 中進行感測器數據的讀取與應用（例如距離感測器、攝影機）。 A:

    • 距離感測器：用於即時測量與障礙物的距離，常用於避障與環境偵測。

    • 攝影機：用於擷取影像並進行影像處理或物件辨識，適合複雜環境監控與自動化任務。

    這些感測器數據可直接應用於機器人控制，實現自主導航、環境感知與智慧應用。

  11. 若要延伸 Tutorial2 讓機器人具備自主避障能力，你會怎麼設計與實作？A:優先使用反應式控制確保即時避障，再疊加規劃層實現高效導航。測試時應逐步增加環境複雜度，從空曠場地到多障礙迷宮，最後加入動態物體驗證。
  12. 請說明 Webots 中「時間步長（timestep）」的意義，以及它對模擬的影響。A:時間步長是 Webots 模擬的核心參數，影響著模擬的精確度、速度與穩定性。設定時需根據模擬需求與硬體效能，在精確度與速度之間取得平衡。
  13. 如何在 Webots 中為機器人新增感測器？請說明基本步驟。A:新增感測器的步驟包括在場景樹中新增感測器節點、調整位置與方向、儲存並測試，最後在控制器程式中啟用與讀取感測器數據。這些步驟適用於大多數標準感測器，如距離感測器、攝影機等。
  14. 請說明如何將 Webots 控制器程式從 Python 改為 C 語言，並指出相關設定。A:

    (1)在 Webots 選單中選擇 Wizards → New Robot Controller

    (2)選擇語言為 C

    (3)命名控制器（例如 my_controller）

    (4)勾選 Open controller in text editor 後點擊 Done

  15. Tutorial1、2 中，若控制器程式有語法錯誤時，Webots 會如何反應？學員該如何排除？A: 

    • 檢查主控台訊息：Webots 會將所有錯誤訊息（包括標準輸出與錯誤）導向主控台，方便學員進行除錯23。

    • 檔案位置與命名：確保控制器檔案位於專案目錄的 controllers 資料夾下，且目錄與檔案命名正確1。

    • 逐步修正：先修正第一個錯誤，再逐步檢查後續問題，避免一次性修正多處錯誤導致混淆。

  16. 如何將 Webots 世界（world）或機器人設定檔分享給他人，確保對方可順利開啟模擬？A:只要按照標準檔案結構打包專案、確認所有依賴資產與控制器皆在目錄內，並附上簡要說明，即可順利分享 Webots 世界或機器人設定檔，讓對方開啟模擬時不會遇到檔案遺漏或路徑錯誤。
  17. 如果想讓機器人在模擬時產生聲音或播放音樂，Webots 提供哪些方法？A:

    新增 Speaker 節點：在機器人或環境物件的 children 子節點下新增 Speaker 節點。

    設定名稱與位置：設定 name 屬性（如 "speaker"）與 translation 屬性，以決定聲音來源位置。

    選擇聲音檔：可於 Speaker 節點設定播放的 .wav 檔，或直接透過控制器程式指定播放內容12。

  18. 請說明如何在 Webots 中同時操作多個機器人，並讓它們各自執行不同控制器。A:

    透過正確設定場景樹名稱、啟動參數和動態識別機制，Webots 可同時運行多個獨立控制的機器人。關鍵在於：

    1. 確保每個機器人有唯一名稱和 <extern> 控制器

    2. 啟動控制器時明確指定 --robot-name

    3. 在程式碼中根據名稱分支執行不同邏輯

    4. 資源使用需隔離（檔案、網路埠等）

    此方法適用於學術研究（如多機器人協作）和工業測試（如自動化產線模擬）。

  19. 在 Webots 控制器中，如何獲得機器人的當前座標位置與朝向？A:

    • 一般控制器：需安裝 GPS 與 Compass 節點，透過感測器讀取位置與朝向。

    • Supervisor 控制器：可直接存取場景樹欄位，無需額外感測器，即可取得精確座標與朝向。

    這種設計讓一般控制器更貼近真實機器人（需依賴感測器），而 Supervisor 則可用於實驗監控與自動化測試。

  20. 請說明 Webots 控制器中感測器資料的單位（例如距離感測器、加速度計），以及如何正確解讀。A:
    感測器類型 回傳值單位 解讀方式與注意事項
    距離感測器 感測器原始輸出 需用 lookupTable 轉換為實際距離（公尺）
    加速度計 公尺每平方秒（m/s²） 三軸加速度，含重力分量
  21. 如果想要錄製 Webots 模擬畫面並輸出成影片檔，該如何操作？A: ShareX.exe
  22. Webots 中的 PROTO 文件有什麼作用？與 world 檔（.wbt）的關係為何？A:
    特性 PROTO 文件 World 文件（.wbt）
    角色 定义可重用组件（如机器人模型） 描述整个模拟场景（包含实例化的 PROTO）
    语法结构 包含 PROTO 声明、字段定义和节点主体 包含 WorldInfo、Viewpoint 和实例化节点
    实例化方式 在 world 文件中通过名称和字段参数实例化：
    MyRobot { translation 0 1 0 }    		 直接包含场景根节点和实例化对象
    依赖关系 可嵌套引用其他 PROTO（需声明 EXTERNPROTO） 依赖 PROTO 文件定义组件
  23. 請說明如何在 Webots 中設定機器人的初始位置和方向。A:

    • 場景樹設定：適用於模擬初始設定，操作簡單直觀。

    • Supervisor 控制器設定：適用於程式化、動態調整機器人初始位置與方向。

    這兩種方式可依需求靈活選擇。

  24. 在 Tutorial1 或 Tutorial2 中，如何讓機器人執行預設外的自訂動作？A:讓機器人執行預設外的自訂動作，關鍵在於編寫自訂控制器程式，並根據機器人結構與需求設計動作邏輯。Webots 支援多種語言（如 Python、C、C++、Java 等），並提供豐富的 API 控制馬達、感測器與其他設備，讓你能靈活實現各種自訂動作。
  25. Webots 支援哪些主流程控制語言？各有何優缺點？A:
    語言 說明與特點 優點 缺點
    C/C++ 編譯型語言，執行效率高，Webots 核心 API 以 C 實現 執行速度快、資源消耗低、適合複雜與即時控制、API 最完整 開發門檻較高、記憶體需自行管理、跨平台需額外編譯環境
    Java 編譯型語言，運行於 JVM 跨平台、易於維護、垃圾回收機制、適合大型專案 執行速度略慢於 C/C++、需安裝 JRE/JDK
    Python 直譯型語言，語法簡潔易上手 開發速度快、易於除錯、適合原型設計與教學 執行速度較慢、不適合計算密集型任務
    MATLAB 數學運算與模擬常用，需安裝 MATLAB 數學運算強大、適合演算法開發與數據分析 需授權、執行速度慢、僅限於 MATLAB 環境
  26. 機器人控制器程式如何與其他程式（如資料記錄、遠端伺服器）進行通訊？A:

    Webots 控制器程式與其他程式通訊的原理，主要是透過進程分離與網路協定，讓控制器能與外部程式（如資料記錄、遠端伺服器）進行資料交換。常見方式包括 Socket 通訊、Emitter/Receiver、檔案/資料庫存取、ROS/ROS2 整合、Web API 等，每種方式各有其適用場景與優點。這種設計讓 Webots 能靈活應用於各種機器人模擬與協作場景。

  27. Webots 的模擬世界中燈光和陰影設定如何影響感測器或相機的數據？A:

    Webots 中的燈光與陰影設定透過物理模型直接影響感測器數據：

    • 光感測器：輻照度計算包含直接/環境光分量，受遮蔽、衰減、濾鏡調控。

    • 攝影機：光照強度決定影像亮度，陰影對比與動態模糊影響影像品質。
      合理配置光源參數與感測器設定，可平衡模擬真實性與運算效能。

  28. 如何在 Webots 中調整機器人或感測器的物理參數（如質量、摩擦係數）？A:Webots 可透過場景樹或 Supervisor 控制器，靈活調整機器人與感測器的物理參數，實現更真實或實驗性的模擬效果。
  29. 請說明如何使用 Webots 的 Console 與 Debug 工具協助程式錯誤排除。A:Webots 的 Console 與 Debug 工具可協助開發者快速定位程式錯誤，無論是語法錯誤、執行異常或資料流問題，都能透過 Console 輸出、gdb 除錯或 ROS 工具進行有效排除。
  30. Tutorial1、2 裡的機器人遇到物體時，碰撞偵測機制如何運作？A:Webots 的碰撞偵測機制基於 boundingObject 與 ODE 物理引擎，自動偵測所有碰撞並模擬真實物理反應。開發者可透過場景樹設定碰撞體，並在必要時以 Supervisor API 進行進階碰撞事件監控與處理。
  31. 若要將 Webots 模擬結果導出並進行數據分析，你會選擇哪些方法與工具？A:
    方法/工具 適用場景 優點
    CSV/JSON 檔案 簡單數據記錄、離線分析 易於導入、跨平台
    ROS Topic/rosbag ROS 整合、即時數據記錄與回放 標準化、支援多節點
    Web Scene 3D 互動展示、線上分享 視覺化、易於分享
    Python 生態系 數據分析、繪圖、機器學習 靈活、資源豐富
    MATLAB/R 專業數學運算、統計分析 功能強大、適合研究
    雲端平台 大規模數據分析、協作 可擴展、支援分散式運算
